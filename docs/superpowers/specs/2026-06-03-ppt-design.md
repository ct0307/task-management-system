# 课程答辩 PPT 设计

**日期**: 2026-06-03
**状态**: 已确认

## 要求

- 时长：5 分钟
- 风格：现代设计型（深色/渐变背景，图标丰富）
- 输出：12 页 .pptx

## 页面结构

### Part 1：项目创意与亮点（P3-P4）

**P3 设计初衷**
- 痛点：大学生管理任务+课程表+倒数日，现有工具太重或太散
- 动机："做一个自己每天都会用的系统"

**P4 核心亮点（3 卡片）**
- 课程表智能解析（OCR + AI）
- 本地文件解析服务（MCP 双协议）
- 轻量化全栈（Docker 一行部署）

### Part 2：项目功能与特色（P5-P7）

**P5 功能模块总览（4 卡片）**
- 任务管理 / 日程规划 / 倒数日 / 系统功能

**P6 项目架构**
- 三层架构图：React → Express → MySQL + Python 解析服务

**P7 技术特色**
- JWT 认证、Zustand 状态管理、批量 API、防抖搜索、响应式布局

### Part 3：开发难点与解决方案（P8-P10）

**P8 难点1：状态管理混乱**
- 双状态库 → 统一 Zustand，修筛选异步 Bug

**P9 难点2：课程表智能解析**
- OCR 链路：pdfplumber → pytesseract → MCP 服务化

**P10 难点3：巨型组件重构**
- TaskList 818 行 → 拆 3 组件，单文件 ≤250 行

### Part 4：AI 开发学习心得（P11-P12）

**P11 学习收获**
- AI 辅助编程效率、MCP 实战、完整开发流程

**P12 不足与展望**
- 移动端、测试覆盖、WebSocket 实时通知

## 视觉规范

- 深色背景 + 渐变装饰
- 卡片式布局
- 图标点缀（emoji 或形状）
- 代码片段用深色背景框
